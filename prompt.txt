로컬 시스템에서 파일들의 정보와 파일 자체를 수집하는 도구를 만들려고해. 아래와 같은 요구사항을 지켜서 작성해줘


개발환경: uv 를 이용한 파이썬 환경 (uv init, uv venv, uv add 로 관리)

요구사항: 파일들의 종류를 확인할 수 있는 라이브러리와 기본적은 정보 수집이 가능해야 함. 주로, Pe 파일과 elf 에 대해서 중점적으로 할 예정. 하지만 다른 파일들도 타입 정보는 확인할 수 있어야됨. 고유 값은 해시를 이용할 건데, md5, sha256을 공통적으로 계산. 가능하다면 파일별 엔트로피도 추출.


명령줄 인자

1. 경로 (절대경로, 상대경로도 가능해야 하고, 운영체제와 무관하게 동작해야 함.)

2. 파일수집플래그 (기본으로 활성화 되며, 경로는 현재 스크립트의 실행경로에 binaries 를 만들어서 보관)

3. 실행파일만 할지 전체로 할지 플래그 (기본은 실행파일만. 실행파일에는 pe, elf 가 포함된다)

3. 정보 저장 방식 (기본으로 csv, sqlite도 지원가능하게, 스크립트와 같은 경로에 database 라는 이름으로 사용)


프로그램 로직 개요

1. 전달받은 인자의 모든 경로의 파일경로를 수집한다. (오래 걸릴시 비동기도 고려)

2. tqdm 과 같은 프로세스 상태를 볼 수 있는 도구로 진행도 표시

3. 각 파일마다 파일 타입을 확인하고, 파일 타입에 따라 파일 정보를 수집한다.

4. 파일 경로, 파일명, 파일 타입, md5, sha256은 기본으로 수집한다.
  - sqlite 일 경우 기본 인덱스는  sha256으로 한다.

5. 파일 수집 옵션이 꺼지지 않으면 binaries 폴더에 sha256으로 파일명을 바꿔서 저장한다. 이 때, 하위 폴더로 앞 2글자를 써서 만들고 그 밑에 위치시킨다.

6. PE 파일은 pefile 라이브러리로 추가 정보를 수집하고, elf 파일은 그에 맞는 라이브러리를 사용한다.

7. 수집이 끝나면 수집 요약을 표시한다.
